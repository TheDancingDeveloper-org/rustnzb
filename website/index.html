rustnzb takes an NZB and does the rest — pipelined NNTP downloads, SIMD yEnc decoding, PAR2 repair, and archive extraction, wrapped in a web UI with real automation. One static binary, written in Rust, built for self-hosters.

215 list of capabilities follows

Engine

pipelining
Multiple ARTICLE commands in flight per connection. Configurable depth per server removes round-trip latency from the hot path.
failover
Priority-ordered server pools over TLS (rustls). Articles missing on one server are retried on the next; fill servers are only touched when needed.
yenc
SIMD-accelerated yEnc decoding with CRC32 validation, streamed straight into the file assembler.
par2
Verify and repair in pure Rust — no external par2 binary, no subprocesses. Damaged files are rebuilt from recovery blocks automatically.
extraction
RAR, 7z, and ZIP archives are unpacked after repair, multi-part sets included, then cleaned up and moved to their category folder.

Interface

web ui
Queue, history with per-download insights, lifetime statistics, live logs, drag-and-drop NZB upload, and selectable themes.
automation
RSS feeds with regex rules, a watch folder, and a newsgroup browser with header search and threaded views.
media
A WebDAV media library that streams files on demand, straight from Usenet, without downloading first.
api
Full REST API with Swagger docs — and a drop-in compatible download-client API, so Sonarr, Radarr, and Prowlarr connect with their standard settings.
anywhere
Docker image, Debian packages, static Linux binaries, a Windows installer, and a Tauri desktop app with a system tray.

220 article body follows — the pipeline

  1. 01 PARSE The NZB's XML becomes segments, groups, and message-IDs. Passwords are read from <meta> tags.
  2. 02 FETCH Pooled TLS connections pull articles with pipelined ARTICLE commands, failing over across servers by priority.
  3. 03 DECODE yEnc bodies are decoded with SIMD, validated against CRC32, and assembled into complete files as they arrive.
  4. 04 REPAIR PAR2 verifies every file; missing blocks are reconstructed in-process by the pure-Rust repair engine.
  5. 05 UNPACK RAR, 7z, and ZIP archives are extracted, leftovers are cleaned up, and the result lands in your category folder.

Docker

$ docker run -d --name rustnzb \
    -p 9090:9090 \
    -v ./config:/config \
    -v ./data:/data \
    -v /path/to/downloads:/downloads \
    ghcr.io/ausagentsmith-org/rustnzb:latest

Then open http://localhost:9090 and add your NNTP servers in the web UI.

Binary

$ tar -xzf rustnzb-v1.3.4-linux-x86_64.tar.gz
$ cd rustnzb-v1.3.4
$ ./rustnzb --port 9090

Debian users: apt install ./rustnzb-*.deb installs a systemd service. Windows users: run the installer.
